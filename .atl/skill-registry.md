## Compact Rules (inject into sub-agent prompts)

### Rule: Modern CLI Tools (ALWAYS)
- Use `bat` (not `cat`), `eza` (not `ls`), `fd` (not `find`), `rg` (not `grep`), `sd` (not `sed`).
- Install via `brew install {tool}` if missing.

### Rule: Commits (ALWAYS)
- Conventional commits only.
- NEVER add `Co-Authored-By`, `🤖 Generated with`, or any AI attribution line.
- Never skip hooks (`--no-verify`) unless user explicitly asks.
- Never amend published commits.
- Destructive git ops (`reset --hard`, `push --force`, `branch -D`) need explicit user approval.

### Rule: Spanish tone (ALWAYS)
- Peruano neutro, tuteo (tú/tienes/puedes/sabes/quieres/arranca/ponte).
- NEVER voseo (vos/tenés/podés/sabés/querés/arrancá/dale/ponete).
- In chat: Peruvian slang (pata/causa/al toque/chamba) OK when fits naturally.
- In docs/commits/public artifacts: pan-Latin flavor (bueno/genial/enseguida/trabajo), NO Peruvian slang.
- Applies to: chat, docs, commits, exercise statements, code comments, sub-agent prompts.

### Rule: Runtime (ALWAYS for this repo)
- Stack: Bun 1.3+ / TypeScript 5.9 strict / `@langchain/core@^1.1`.
- Run `bun`/`bunx` commands from `code/` directory — NOT repo root.
- Typecheck: `bunx tsc --noEmit` (do not "build" after changes).
- Tests: `bun test` (bun:test framework; files `*.test.ts` co-located with sources).

### Rule: Exercise Contract (when touching exercises/)
- Each exercise: `meta.json` (with `locales: ["es","en"]` minimum), `starter.ts`, `solution.ts`, `tests.test.ts`, `{locale}/exercise.md` per declared locale.
- Tests use `resolveExerciseFile(import.meta.url)` + `LCDEV_TARGET` env (`starter|solution`).
- Asserts are LangChain-level (capture via `BaseChatModel` patch), NOT SDK-level.
- Exercise exports `default async function run()` — harness imports + invokes it.
- Spanish exercise statements: peruano neutro tuteo.

### Rule: SDD Artifact Store (this repo)
- Mode: **hybrid** (files in `openspec/` + engram mirror at `sdd/{change}/{artifact-type}`).
- User preference: DO NOT default to pure `engram` — always include file-backed artifacts.
